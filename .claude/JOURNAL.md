23. **Task - Add dynamic time ago indicator** (v1.2.13): Inject live-updating relative timestamp into notification toast and notification center DOM<br>
    **Result**: Created `src/utils.ts` with `formatTimeAgo(createdAt)` returning compact relative time labels across 5 tiers (just now, Xs ago, Xm ago, Xh ago, Xd ago) and `appendTimeAgo(message, createdAt)` helper. Implemented DOM injection approach with two pathways: `injectTimeAgo()` handles toast popups by finding the matching `.jp-toast-message` element via 100ms `setTimeout` after `notify()` returns, while `observeNotificationCenter()` uses a `MutationObserver` on `document.body` watching for `.jp-Notification-Center` to appear and for list items added inside an existing center - when detected, `injectTimeAgoIntoCenter()` matches DOM elements to `Notification.manager.notifications` entries by message text to look up `createdAt` timestamps. Both pathways append a styled `<div class="jp-toast-time-ago">` with 0.75em font and `--jp-ui-font-color2` color, refreshing every 10 seconds. When action buttons are present (`.jp-toast-buttonBar`), the time-ago element is inserted as the first child of the button bar so it appears inline on the same row as buttons - left-aligned while buttons stay right. When no buttons exist, it appends below the message text. Toast intervals self-clear via `manager.has(id)`, center intervals self-clear when center is removed from DOM. Initial approach of appending text to message string was replaced because separate DOM elements are visually cleaner and avoid `Notification.manager.update()` reordering the notification stack. Notification center injection required `setTimeout` delay because list items render asynchronously after the center container is added to the DOM. Verified via Playwright MCP that toast DOM injection works correctly. Created `src/__tests__/utils.spec.ts` with 7 Jest test cases. Fixed TypeScript `TS2488` by wrapping `querySelectorAll` with `Array.from()` for ES2018 target. Added two Galata integration tests in `ui-tests/tests/jupyterlab_notifications_extension.spec.ts`: one verifying `.jp-toast-time-ago` element appears on notifications with valid time format, another verifying time-ago is placed inside `.jp-toast-buttonBar` alongside action buttons. Both POST notifications via the ingest API and wait for the polling cycle. Updated README with Time-Ago Indicator section and added feature to Key Features list. Also created workspace-level Playwright skill at `.claude/skills/playwright/SKILL.md` documenting JupyterHub OAuth bypass, JupyterLab CSS selectors, and DOM inspection patterns.

24. **Task - Fix cumulative time-ago duplication** (v1.2.15): Fix MutationObserver cascade adding duplicate `.jp-toast-time-ago` elements in notification center<br>
    **Result**: Fixed bug where `injectTimeAgoIntoCenter()` was cumulatively appending hundreds of `.jp-toast-time-ago` elements inside `.jp-toast-buttonBar`. Root cause: the duplicate guard `el.querySelector('.jp-toast-time-ago')` only checked inside `.jp-toast-message`, but when action buttons exist the time-ago element is inserted into the sibling `.jp-toast-buttonBar` - so the guard never found it. Each MutationObserver callback added another element, which triggered another DOM mutation, creating a cascade. Fixed by hoisting `elParent` declaration and checking `elParent.querySelector('.jp-toast-time-ago')` which covers both cases - time-ago inside the message div and time-ago inside the sibling button bar. Published v1.2.15 to npm and PyPI.

25. **Task - Fix time-ago message matching and timestamps** (v1.2.20): Fix multiline message matching and notification center timestamp consistency<br>
    **Result**: Fixed three related issues in time-ago injection. First, notifications containing newlines (e.g. `"Idle resources culled:\nTerminals: 1"`) failed to match because JupyterLab renders `\n` as `<br>` in the DOM and `textContent` drops the linebreak entirely, producing `"Idle resources culled:Terminals: 1"` which doesn't match the stored message. Added `normalizeMsg()` at module level collapsing all whitespace to single spaces before comparison, applied to both `injectTimeAgo()` toast matching and `injectTimeAgoIntoCenter()` center matching. Second, notification center showed different time-ago than the toast because `injectTimeAgoIntoCenter()` was using `Notification.manager.notifications[i].createdAt` (set when `notify()` was called in the browser at page load) instead of the server-side `createdAt` from the original notification payload. Added module-level `serverCreatedAtMap` populated during `fetchAndDisplayNotifications` when toasts are created, used as primary lookup in center injection with JupyterLab manager timestamps as fallback for non-extension notifications. Also fixed center refresh interval which was trying to reconstruct message text from `parent.childNodes` (broken when time-ago lives in button bar) - switched to reading `data-created-at` attribute stored directly on the time-ago element by `createTimeAgoElement()`. Third, `textContent` was still not matching multiline messages because `<br>` produces zero characters in `textContent` while `\n` normalizes to a space - switched to `innerText` which respects `<br>` elements and returns `\n` for them, making normalization produce identical results for both DOM and stored message strings. Published v1.2.20 to npm and PyPI.
